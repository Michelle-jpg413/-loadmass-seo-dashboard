Connect Google Search Console

Takes about 5 minutes. You'll get exact ranking positions, impressions, and click data pulled live from your GSC property.

1
Open Google Cloud Console

Go to console.cloud.google.com and create a new project (or select an existing one).

2
Enable the Search Console API

In the project, go to APIs & Services → Library, search for "Google Search Console API", and click Enable.

3
Create OAuth credentials

Go to APIs & Services → Credentials → Create Credentials → OAuth client ID. Choose Web application.

How this was put together

The keyword signals in the Rankings tab come from two sources: (1) the original research snapshot taken 25 Jun 2026, which checked page-one results for each term; and (2) live Google Search Console data when connected, which adds real average-position, impressions and click data directly from Google for loadmass.com.au.

GSC position matching works by checking whether any GSC query contains or closely matches the tracked keyword term. Where no GSC match is found, the cell shows "—" — meaning Google has not recorded measurable impressions for that term (confirms the Gap signal).

GSC setup & troubleshooting

If you see a "popup blocked" message when connecting, allow popups for this page and try again. If you get an OAuth error, make sure the exact origin shown in the setup screen is listed under Authorised JavaScript origins in your Google Cloud Console credentials — it must match character-for-character. Tokens expire after 1 hour; clicking the badge again will silently renew.
